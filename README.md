# Интерпретатор подмножества языка Go, написанный на Go с использованием парсера ANTLR4.

## Запуск

### Docker

```bash
docker build -t goterp .
docker run --rm -v $(pwd)/examples:/examples goterp ./bin/goterp /examples/showcase.go
```

### Сборка

```bash
make build
./bin/goterp examples/showcase.go
```

## Использование

```bash
goterp <файл.go>
```

При запуске без аргументов выводится краткая справка (exit code 1).

## Поддерживаемый язык

### Структура программы

Программа начинается с `package main`, за которым следуют объявления типов
и функций в произвольном порядке. Входная точка - функция `main`.
Каждый оператор завершается `;` (точка с запятой обязательна).

```go
package main
func main() {
  fmt.Println("Hello, World!");
}
```

### Типы данных

| Тип | Описание | Нулевое значение |
|-----|----------|------------------|
| `int` | 64-битное целое | `0` |
| `float64` | 64-битное вещественное | `0` |
| `string` | строка | `""` |
| `bool` | логическое | `false` |
| `[]T` | срез элементов типа T | пустой срез |
| `type S struct {...}` | пользовательская структура | все поля = ноль |

### Переменные

```go
var x int = 10;       // объявление с типом
var f float64;        // нулевое значение
y := 20;              // краткое объявление (тип выводится)
x = x + 1;            // присваивание
```

Повторное объявление в той же области видимости - ошибка.
Затенение переменной во вложенном блоке разрешено.

### Выражения и операторы

Арифметика: `+`, `-`, `*`, `/`, `%`  
Сравнения: `<`, `<=`, `>`, `>=`, `==`, `!=`  
Логика: `&&`, `||`, `!`  
Унарный минус: `-expr`

Приоритеты соответствуют Go. Смешение типов (`int + float64`) - ошибка.

### Вывод

```go
fmt.Println(a, b, c);
fmt.Printf("%d + %d = %d\n", 1, 2, 3);
```

### Условные операторы

```go
if cond {
  // ...
} else if cond2 {
  // ...
} else {
  // ...
}
```

Условие должно быть типа `bool`.

### Циклы `for`

```go
for i := 0; i < 10; i = i + 1 { ... }   // с init/cond/post
for x < 10 { ... }                      // только условие (while-форма)
for { ... }                             // бесконечный (нужен break)
```

`break` - выход из цикла, `continue` - следующая итерация.
Использование `break`/`continue` вне цикла - ошибка.
Бесконечный цикл обрывается по достижении лимита итераций.

### Срезы `[]T`

```go
s := []int{1, 2, 3};
var empty []int;
fmt.Println(len(s));
fmt.Println(s[0]);
s[1] = 10;
s = append(s, 4);
```

Итерация:

```go
for i, v := range s { ... }
for i := range s { ... }
```

Выход за пределы среза - ошибка времени выполнения.

### Структуры

```go
type Point struct {
  x int;
  y int;
}

p := Point{x: 1, y: 2};  // литерал (неуказанные поля = ноль)
var q Point;             // ноль
fmt.Println(p.x);        // чтение
p.x = 5;                 // запись
```

> Ограничение: struct и срез имеют ссылочную семантику (две переменные
> могут указывать на одно и то же значение). Это отличается от поведения
> настоящего Go, где struct копируется при присваивании.

### Функции

```go
func add(a int, b int) int { return a + b; }
func swap(a int, b int) (int, int) { return b, a; }
func greet(name string) { fmt.Println("Hello", name); }

result := add(3, 4);
x, y := swap(1, 2);
greet("World");
```

Рекурсия поддерживается. Функция не видит переменные вызывающего кода.

## Коды возврата

| Код | Ситуация |
|-----|----------|
| 0 | Программа выполнена успешно |
| 1 | Нет аргументов / не удалось открыть файл |
| 2 | Лексическая или синтаксическая ошибка |
| 3 | Семантическая ошибка (необъявленная переменная, несоответствие типов и т.д.) |
| 4 | Ошибка времени выполнения (деление на ноль, выход за границы среза, перебор итераций) |

## Ограничения

- Точка с запятой обязательна в конце каждого оператора
- Только `for range` по срезам (не по map, string)
- Нет `make`, `copy`, `new`, sub-slicing (`s[1:3]`)
- Нет вложенных полей в левой части присваивания (`p.a.b = v` не поддерживается)
- Нет `map`, каналов, горутин, интерфейсов, методов на типах
- Нет `import` (только `fmt.Println`/`fmt.Printf` встроены)
- Ссылочная семантика struct и slice
