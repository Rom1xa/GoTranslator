# GoSubset Interpreter

Интерпретатор подмножества языка Go, написанный на Go с
использованием ANTLR4.

## Запуск

### Сборка

```bash
make build
./bin/goterp examples/showcase.go
```

### Docker

```bash
docker build -t goterp .
docker run --rm -v $(pwd)/examples:/examples goterp ./bin/goterp /examples/showcase.go
```

### Формат CLI

```bash
goterp <файл.go>
```

## Типы данных

| Тип                        | Описание                     | Нулевое значение        |
| -------------------------- | ---------------------------- | ----------------------- |
| `int`                      | 64-битное целое число        | `0`                     |
| `float64`                  | 64-битное вещественное число | `0`                     |
| `string`                   | строка                       | `""`                    |
| `bool`                     | логическое значение          | `false`                 |
| `[]T`                      | срез элементов типа `T`      | пустой срез             |
| `type S struct { ... }`    | пользовательская структура   | нулевые значения полей  |
| `type I interface { ... }` | интерфейсный тип             | `nil`-подобное значение |

Язык статически и строго типизирован. Например, `1 + 2.5` является ошибкой,
потому что `int` и `float64` не приводятся друг к другу автоматически.

## Объявления и области видимости

Поддерживаются обычные объявления, краткие объявления и присваивания.

```go
var x int = 10;
var y int;
z := 20;
x = x + z;
```

Повторное объявление имени в той же области видимости считается семантической
ошибкой. Затенение во вложенном блоке разрешено:

```go
x := 1;
{
  x := 2;
  fmt.Println(x); // 2
}
fmt.Println(x);   // 1
```

Каждый блок `{ ... }` создает новую область видимости. Вызов функции создает
отдельный кадр: функция видит только свои параметры, локальные переменные и
глобально зарегистрированные функции/типы, но не локальные переменные
вызывающего кода.

## Выражения

Поддерживаются:

- литералы `int`, `float64`, `string`, `true`, `false`;
- идентификаторы;
- арифметика `+`, `-`, `*`, `/`, `%`;
- сравнения `<`, `<=`, `>`, `>=`, `==`, `!=`;
- логические операции `&&`, `||`, `!`;
- унарный минус `-expr`;
- вызовы функций;
- доступ к полям `p.x`;
- индексация `s[i]`;
- литералы структур и срезов.

Приоритет операторов соответствует привычному порядку Go: унарные операции
сильнее умножения, умножение сильнее сложения, сравнения слабее арифметики,
`&&` сильнее `||`.

## Вывод

Импортов в языке нет.Вместо `import "fmt"` интерпретатор имеет встроенную
поддержку двух функций:

```go
fmt.Println(a, b, c);
fmt.Printf("%d + %d = %d\n", 1, 2, 3);
```

`fmt.Printf` требует, чтобы первый аргумент был строкой формата.

## Условия

```go
if x > 0 {
  fmt.Println("positive");
} else if x == 0 {
  fmt.Println("zero");
} else {
  fmt.Println("negative");
}
```

Условие `if` должно иметь тип `bool`. Числа, строки и другие значения не
приводятся к `bool`.

## Циклы

Поддерживаются три формы обычного `for`.

```go
for i := 0; i < 10; i = i + 1 {
  fmt.Println(i);
}

for x < 10 {
  x = x + 1;
}

for {
  break;
}
```

`break` завершает ближайший цикл, `continue` переходит к следующей итерации.
Использование `break` или `continue` вне цикла является семантической ошибкой.

## Функции

Функции объявляются только на верхнем уровне.

```go
func add(a int, b int) int {
  return a + b;
}

func swap(a int, b int) (int, int) {
  return b, a;
}

func greet(name string) {
  fmt.Println("Hello", name);
}
```

Поддерживаются параметры, отсутствие возвращаемого значения, одно возвращаемое
значение, несколько возвращаемых значений и рекурсия.

```go
result := add(3, 4);
x, y := swap(1, 2);
greet("World");
```

Если функция объявлена с возвращаемыми типами, она должна вернуть значения
соответствующего количества и типов.

## Структуры

```go
type Point struct {
  x int;
  y int;
}

func main() {
  p := Point{x: 1, y: 2};
  var q Point;

  fmt.Println(p.x);
  p.x = 10;
  fmt.Println(q.x);
}
```

В литерале структуры можно указать часть полей. Неуказанные поля получают
нулевые значения. Обращение к несуществующему полю является семантической
ошибкой.

Важное отличие от Go: структуры в GoSubset имеют ссылочную семантику.

```go
p := Point{x: 1, y: 2};
q := p;
q.x = 100;
fmt.Println(p.x); // 100 в GoSubset
```

В настоящем Go присваивание структуры копирует значение, поэтому аналогичный
код напечатал бы `1`. В GoSubset переменные `p` и `q` указывают на один и тот
же объект структуры.

## Срезы

```go
s := []int{1, 2, 3};
var empty []int;

fmt.Println(len(s));
fmt.Println(s[0]);

s[1] = 10;
s = append(s, 4);
```

Поддерживаются:

- литералы `[]T{...}`;
- пустой срез как нулевое значение;
- `len(s)`;
- `append(s, value)`;
- чтение по индексу `s[i]`;
- запись по индексу `s[i] = value`;
- `for range` по срезу.

```go
for i, v := range s {
  fmt.Println(i, v);
}

for i := range s {
  fmt.Println(i);
}
```

Индекс должен иметь тип `int`. Выход за границы среза является ошибкой времени
выполнения.

Срезы также имеют ссылочную семантику: несколько переменных могут указывать на
один и тот же срез. При этом `append` возвращает новое значение среза, поэтому
результат нужно присваивать явно:

```go
s = append(s, 4);
```

## Интерфейсы

Поддерживается синтаксис объявления интерфейсных типов:

```go
type Any interface {
}

type Printer interface {
  Print(prefix string);
  // ...
}
```

Интерфейсная переменная хранит динамическое значение:

```go
type Any interface {
}

func main() {
  var x Any = 10;
  fmt.Println(x);

  x = "text";
  fmt.Println(x);
}
```

Текущая реализация интерфейсов минимальная. Интерфейс работает как именованный
контейнер для динамического значения. Полноценная проверка method set не
реализована, потому что в диалекте пока нет методов на типах.

## Горутины

Поддерживается минимальный оператор `go` для запуска пользовательской функции
или встроенного вывода.

```go
func say(msg string) {
  fmt.Println(msg);
}

func main() {
  go say("from goroutine");
  go fmt.Println("parallel print");
}
```

Аргументы `go`-вызова вычисляются сразу в текущем потоке интерпретатора, после
чего функция запускается как фоновая Go-горутина интерпретатора. Перед завершением
`Run` интерпретатор ждет окончания всех запущенных горутин ( WaitGroup ), чтобы программа не
теряла вывод и тесты были воспроизводимыми.

Вывод из горутин синхронизируется. Если фоновая функция завершилась ошибкой,
интерпретатор возвращает эту ошибку после завершения `main`.

## Встроенные функции

| Функция                       | Назначение                           |
| ----------------------------- | ------------------------------------ |
| `fmt.Println(args...)`        | печать аргументов с переводом строки |
| `fmt.Printf(format, args...)` | форматированный вывод                |
| `len(s)`                      | длина среза или строки               |
| `append(s, value)`            | новый срез с добавленным элементом   |

Другие функции стандартной библиотеки Go не поддерживаются.

## Отличия от настоящего Go

Основные отличия:

- обязательная точка с запятой после каждого оператора;
- нет автоматической вставки `;`;
- нет `import`, пакетов и стандартной библиотеки, кроме встроенных
  `fmt.Println` и `fmt.Printf`;
- нет `const`;
- нет указателей;
- нет методов на типах;
- нет `map`;
- нет каналов и `select`;
- нет `defer`, `panic`, `recover`;
- нет `switch`;
- нет `fallthrough`;
- нет `goto`;
- нет `make`, `new`, `copy`;
- нет срезовых выражений вида `s[1:3]`;
- нет массивов фиксированной длины;
- нет анонимных функций и замыканий;
- нет методов стандартной библиотеки;
- нет неявных преобразований типов;
- структуры имеют ссылочную семантику, а не value-семантику Go;
- срезы имеют упрощенную модель без `cap`;
- `append` принимает один добавляемый элемент;
- `for range` поддержан только для срезов;
- горутины реализованы минимально

## Диагностика и коды возврата

| Код | Ситуация                                   |
| --- | ------------------------------------------ |
| `0` | программа выполнена успешно                |
| `1` | нет аргументов или не удалось открыть файл |
| `2` | лексическая или синтаксическая ошибка      |
| `3` | семантическая ошибка                       |
| `4` | ошибка времени выполнения                  |

Примеры семантических ошибок: необъявленная переменная, повторное объявление в
одной области, несовпадение типов, неверное количество аргументов функции,
`break` вне цикла.

Примеры ошибок времени выполнения: деление на ноль, выход за границы среза или
строки, превышение лимита итераций.
